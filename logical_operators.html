邏輯與比較運算符號

學習導覽

在本章中，我們將學習 Python 中用於條件判斷的核心工具：**比較運算符號**和**邏輯運算符號**。這些符號是控制程式流程的基礎。

  • **比較運算符號**：用於比較兩個值之間的關係，例如相等、大於或小於。
  • **邏輯運算符號**：用於結合多個布林值（Boolean values），例如 **and**、**or** 和 **not**。

比較運算符號 (Comparison Operators)

比較運算符號總是回傳一個布林值（`True` 或 `False`）。

運算符號 名稱 範例
== 等於 x == y
!= 不等於 x != y
> 大於 x > y
< 小於 x < y
>= 大於等於 x >= y
<= 小於等於 x <= y

範例：

a = 10
b = 5

print(a > b)    # 輸出 True
print(a == b)   # 輸出 False
print(a != b)   # 輸出 True

邏輯運算符號 (Logical Operators)

邏輯運算符號用於結合布林表達式，讓你可以建構更複雜的條件判斷。

運算符號 名稱 描述
and 邏輯與 兩個條件都為 True 時，結果才為 True。
or 邏輯或 任一條件為 True 時，結果即為 True。
not 邏輯非 將布林結果反轉（True 變 False，False 變 True）。

範例：

age = 25
is_student = True

print(age > 18 and is_student)  # 輸出 True，因為 25 > 18 (True) 且 is_student (True)
print(age < 18 or is_student)   # 輸出 True，因為 is_student 為 True
print(not is_student)           # 輸出 False

短路求值 (Short-circuit Evaluation)

Python 在執行邏輯運算符號時採用短路求值。這表示：

  • **對於 `and`**：如果第一個運算對象為 `False`，則不會評估第二個運算對象，結果直接為 `False`。
  • **對於 `or`**：如果第一個運算對象為 `True`，則不會評估第二個運算對象，結果直接為 `True`。

常見問題與解答

問題 1：在 Python 中如何表示「介於」？

Python 支援連鎖比較（Chained Comparison），你可以將多個比較運算符號連接起來：

x = 15
# 檢查 x 是否介於 10 和 20 之間
result = 10 < x < 20  # 相當於 10 < x and x < 20
print(result) # 輸出 True

問題 2：`and` 和 `or` 不僅回傳 `True/False` 嗎？

當 `and` 和 `or` 運算的不是單純的布林值（例如數字或字串）時，它們會回傳運算式中的某個物件本身：

  • **`A and B`**：如果 A 為假，回傳 A；否則回傳 B。
  • **`A or B`**：如果 A 為真，回傳 A；否則回傳 B。
print(5 and 0)     # 輸出 0 （因為 5 是 True，回傳後面的 0）
print("Hello" or 0) # 輸出 'Hello' （因為 "Hello" 是 True，回傳自己）

挑戰題目

挑戰 1：複合條件判斷 🛈

給定兩個變數 `temperature = 28` 和 `is_sunny = True`。撰寫程式判斷是否「溫度大於 25 度 **且** 天氣晴朗」。

# 解答
temperature = 28
is_sunny = True
condition = temperature > 25 and is_sunny
print(f"條件是否成立：{condition}")

挑戰 2：使用 `or` 的篩選 🛈

給定變數 `day = "Sunday"` 和 `is_holiday = False`。撰寫程式判斷是否「今天是星期六 **或** 星期天 **或** 是假日」。

# 解答
day = "Sunday"
is_holiday = False
condition = (day == "Saturday" or day == "Sunday" or is_holiday)
print(f"是否為週末或假日：{condition}")

挑戰 3：`not` 的應用 🛈

給定變數 `is_logged_in = True`。撰寫程式判斷使用者是否「未登入」。

# 解答
is_logged_in = True
is_not_logged_in = not is_logged_in
print(f"使用者未登入：{is_not_logged_in}")

挑戰 4：連鎖比較 🛈

給定變數 `score = 85`。撰寫程式判斷分數是否「大於等於 70 **且** 小於 90」。

# 解答
score = 85
condition = 70 <= score < 90  # 連鎖比較
print(f"分數是否在 70 到 90 之間（不含 90）：{condition}")

挑戰 5：短路求值實踐 🛈

給定變數 `x = 0` 和 `y = 10`。在一個條件判斷中，檢查是否「`x` 不為 0 **且** `y` 除以 `x` 的結果大於 5」。思考短路求值如何避免除以零錯誤。

# 解答
x = 0
y = 10

# 由於 x 為 0（False），and 會短路，不會執行 y / x，避免 ZeroDivisionError
condition = x != 0 and (y / x > 5)

print(f"條件結果：{condition}")

挑戰 6：優先順序與括號 🛈

給定 `a = True`, `b = False`, `c = True`。計算 `a and b or c` 和 `a and (b or c)` 的結果並解釋差異。

# 解答
a = True
b = False
c = True

# 邏輯運算符號的優先順序是 not > and > or
result1 = a and b or c  
# 相當於 (a and b) or c
# (True and False) or True  -> False or True -> True
print(f"a and b or c 的結果：{result1}")

# 使用括號強制改變優先順序
result2 = a and (b or c) 
# True and (False or True) -> True and True -> True
print(f"a and (b or c) 的結果：{result2}")

# 備註：這個範例中結果一樣，但表達式 (True and False) or True 還是 (False or True) 會有差異。
# 如果 a=False, b=True, c=False:
# a and b or c -> (False and True) or False -> False or False -> False
# a and (b or c) -> False and (True or False) -> False and True -> False
# 在此例中，優先順序的影響較小，但一般情況下建議使用括號確保邏輯清晰。

挑戰 7：數字作為布林值 🛈

給定 `num1 = 5` 和 `num2 = 0`。計算 `num1 and num2` 和 `num1 or num2` 的結果。

# 解答
num1 = 5  # True
num2 = 0  # False

# and：如果 num1 是 True，回傳 num2
result_and = num1 and num2
print(f"num1 and num2 的結果：{result_and} (此處回傳 0)")

# or：如果 num1 是 True，回傳 num1
result_or = num1 or num2
print(f"num1 or num2 的結果：{result_or} (此處回傳 5)")

結語

比較運算符號與邏輯運算符號是程式中決策的核心。掌握它們的用法和優先順序，尤其是在複合條件和短路求值方面的知識，對於撰寫健壯且高效的 Python 程式至關重要。
